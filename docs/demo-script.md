# 比赛演示脚本（约 5 分钟）

## 准备

- 真机：已安装 HAP，`docs/device-test-checklist.md` 相应项已通过。
- 无真机（当前状态）：`flutter run -d macos`（需临时加 macos 平台）或桌面
  Debug 运行；应用顶部显示黄色“模拟能力模式”横幅，演示 Mock OCR 全链路。

## 演示样例（合成，无个人信息）

`assets/demo/campus_day.txt` 的内容（真机演示前把它做成截图存入相册）：

```
校园创新体验日
报名截止：7月20日 18:00
活动时间：7月25日 14:00—16:30
地点：大学生活动中心 201
入场码：A7281
```

## 步骤

1. **分享导入**（30s）：图库中长按演示截图 → 分享 → FreshCue。
   应用冷启动直接进入处理页（读取→识别→分析→准备四阶段）。
2. **确认页**（90s）：
   - 指出**一图两时间**被分为「截止 7月20日 18:00」和「活动开始 7月25日 14:00
     （含结束 16:30）」两个语义角色；
   - 点击时间候选，原图上高亮对应 OCR 证据框；
   - 展示年份推断解释文字（“截图未包含年份，按 2026 年理解”）；
   - 底部提醒 chips 预览：“保存后将创建 5 条系统提醒”。
3. **首页**（30s）：卡片显示“截止还有 2 天”，入场码遮罩为 `A•••1`。
4. **真实提醒**（60s）：设置 → 关于（Debug 直接进）→ 诊断 →
   “创建 5 分钟演示提醒”。真机上锁屏等待系统通知（代理提醒，进程可被杀）。
5. **通知行为**（30s）：通知上点“完成” → 卡片进入完成状态。
6. **实况窗**（可选，需测试权益）：详情页 →“开启实况胶囊（实验能力）”，
   展示 5 分钟倒计时；无权益时展示回退提示（这本身是卖点：降级路径完整）。
7. **过期箱**（30s）：演示一张已过期卡片自动归入过期箱，可“恢复并重设时间”。

## 红线

- 不伪造系统通知或实况窗截图；Mock 模式必须保留黄色横幅出镜。
- 演示只改样本和测试时间，不改产品逻辑。
