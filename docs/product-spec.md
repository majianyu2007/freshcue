# 产品规格（摘要）

- 产品：截期 / FreshCue —— 把截图中的临时信息转换成会提醒、会过期的时效卡片。
- 核心概念：Temporal Card（时效卡片）：有语义时间锚点、提醒计划和生命周期的
  结构化临时信息，每个字段可回溯原截图 OCR 证据。

## 与竞品差异

不是 OCR 笔记（有生命周期与提醒）、不是待办（自动从截图结构化）、
不是日历（管理“会过期的信息”而非日程，支持取件码/临时码等非事件类）。

## 六类卡片与默认提醒

| 分类 | 关键字段 | 默认提醒 |
|---|---|---|
| pickup 取件/取餐 | 取件码、地点、失效时间 | 失效前 30 分钟；可选 2 小时后；末段可开实况胶囊 |
| event 活动/会议/考试 | 开始/结束、地点、报名截止 | 开始前 1天/1小时/10分钟；截止前 1天/2小时 |
| ticket 票务/出行 | 出发时间、座位/车次、检票截止 | 前 1天/2小时/30分钟 |
| deadline 截止事项 | 截止时间、提交地点 | 前 1天/2小时/15分钟 |
| temporary_secret 临时码 | 敏感内容、失效时间 | 不响铃；失效前 30 分钟；锁屏隐藏 |
| generic 通用 | 自选保鲜期（今晚/24h/3天/自定义） | 失效前 1 小时 |

高风险信息（身份证/银行卡）：提示不建议保存，日志与锁屏遮罩。

## 生命周期

持久状态：draft → active → completed / archived（删除为硬删除+级联清理）。
派生状态（不落库，FreshnessPolicy 实时计算）：fresh / upcoming(≤24h) /
urgent(≤2h) / expired。

## 提醒规则（§16 全量实现于 ReminderPolicy）

多锚点多 offset；不建过去的提醒；跳过单项不影响其他；同触发时间去重；
编辑重算；安静时段 23:00–07:00 仅调整非紧急（提前量≥12h）并向用户说明；
无默认无限重复；通知含标题+时间语义+距离，敏感遮罩；点击进详情；
延后创建新实例并记录来源。
